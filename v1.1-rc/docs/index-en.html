Abstract

This version of the vocabulary has been generated in the context of the OpenCityData initiative, to support the set of datasets identified in the open data guides of the Spanish Federation of Municipalities and Provinces (FEMP). The purpose of the vocabulary is to represent data related to a cultural agenda.

Vocabulary for the representation of events in a cultural agenda: Description back to ToC

Figure 1 shows the classes and properties of the vocabulary that represent the cultural events that are to take place in a given location:


To represent an event we have the concept esagenda:Evento t for which the classes schema:Event, schema:DigitalDocument, schema:CreativeWork, schema:Place y schema:PostalAddress from the schema.org vocabulary and esequip:Equipment from esequip vocabulary.

In order to implement the information already developed about events in the schema.org vocabulary, it has been established that esagenda:Event is a subclass of schema:Event. On the other hand, to represent that an event must have a digital file associated with it (which can be from a web page or a PDF file), the relationship with the schema:DigitalDocument class has been established through the esagenda:documentation property. Finally, as an event must store information about the place where it will take place, it has been decided to relate esagenda:Event with the schema:Place class through the schema:location property.

In order to represent the equipment where the event may take place, it has been implemented that esequip:Equipment is a subclass of schema:Place. In addition, in order to locate the equipment mentioned above, a relationship has been established through the schema:address property between schema:Place and the schema:PostalAddress class.

Additionally, a SKOS concept scheme called esagenda:event-type has been used in this vocabulary to represent the type of event it can be. The values have been specified in http://vocab.linkeddata.es/datosabiertos/kos/cultura-ocio/agenda.


3.1 Examples of vocabulary use

In order to facilitate the understanding of the use of some classes and properties of this vocabulary, some examples will be provided below.

First, for the representation of an event, an instance of the class esagenda:Evento is created with the following properties schema:isAccessibleForFree (indicates if access is free), schema:eventStatus (indicates the status), schema:eventAttendanceMode (indicates assistance mode), schema:workPerformed (indicates work being performed), schema:doorTime (indicates the time at which access is available), schema:audience (indicates recommended audience), schema:startDate (indicates the date and time at which it starts), schema:endDate (indicates the date and time at which it ends), schema:maximumAttendeeCapacity (indicates the maximum number of people that can attend), schema:typicalAgeRange (indicates recommended age), esagenda:interior (indicates if the site is roofed). It is also linked through the property esagenda:tipoEvento, the event type with the corresponding concept of the event type taxonomy. Finally, the event is related to an instance of esagenda:documentation and esequip:Equipament. 
   <http://vocab.linkeddata.es/datosabiertos/recurso/cultura-ocio/agenda/evento/1285153785025> a esagenda:Evento ;
      schema:audience  "Público familiar" ; 
      schema:startDate "2022-08-12T22:00+01:00"^^xsd:dateTime ;
      schema:endDate "2022-08-13T01:00+01:00"^^xsd:dateTime ;
      schema:eventAttendanceMode "Presencial" ;
      schema:isAccessibleForFree "false"^^xsd:boolean ;
      schema:maximunAttendeeCapacity "301" ;
      schema:eventStatus "Por empezar ;
      schema:typicalAgeRange "0-" ;
      esagenda:tipoEvento <http://vocab.linkeddata.es/datosabiertos/kos/cultura-ocio/agenda/tipo-evento/teatro> ;
      esagenda:documentacion <http://vocab.linkeddata.es/datosabiertos/recurso/cultura-ocio/agenda/evento/1285153785025/documentacion> ;
      esequip:Equipamiento <http://vocab.linkeddata.es/datosabiertos/recurso/urbanismo-infraestructuras/equipamiento-municipal/equipamiento/1284236227322> .

The following is an instance of esagenda:documentation with the following properties dct:identifier (indicates identifier), schema:name (indicate name), schema:description (indicates description), schema:image (indicates promotional image), schema:url (indicates the link where the information can be found), schema:fileFormat (indicates the format where the information is). 
   <http://vocab.linkeddata.es/datosabiertos/recurso/cultura-ocio/agenda/evento/1285153785025/documentacion> a esagenda:Documentacion
       dct:identifier "1285153785025" ;
       schema:name "'Caravana escape room - Houdini experience' (HÉCTOR SANSEGUNDO)" ;
       schema:description "Disfruta de una mezcla perfecta de espectáculo de teatro y magia dentro de un museo inversivo , y forma parte de la historia viviendo un escape room único, original y con sorpresas inolvidables."; 
       schema:image "https://cultura.jcyl.es/web/jcyl/binarios/744/545/CARAVANA%20LA%20MAQUINA.jpeg"^^xsd:anyURI ;
       schema:url "https://cultura.jcyl.es/web/jcyl/Cultura/es/Plantilla100Detalle/1284282053297/Evento/1285153785025/Comunicacion"^^xsd:anyURI ;
       schema:fileFormat "HTML" .

The following is an instance of esequip:Equipment with the following properties dct:identifier (indicates identifier) and esequip:nombre (indicate name). Finally, the equipment is related through the propertyschema:address with an instance of schema:PostalAddress. 
   <http://vocab.linkeddata.es/datosabiertos/recurso/urbanismo-infraestructuras/equipamiento-municipal/equipamiento/1284236227322> a esequip:Equipamiento
      dct:identifier "1284236227322" ;
      esequip:nombre "Centro Cultural de la Villa (Lumbrales)" ;
      schema:address <http://vocab.linkeddata.es/datosabiertos/recurso/cultura-ocio/agenda/evento/1285153785025/address> .

The following is an instance of schema:PostalAddress with the following properties schem:addressRegion (indicate province), schema:addressLocality (indicates municipality), schema:postalCode (indicate postal code), schema:streetAddress (indicates address).
