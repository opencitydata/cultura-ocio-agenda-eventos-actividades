3. Description back to ToC

La siguiente figura muestra el diagrama de clases de la ontología, donde se puede observar que el elemento central es la clase esagenda:Evento, definida en este vocabulario

En la figura también se muestran todas las propiedades definidas en este vocabulario, para representar el lugar o equipamiento de la ciudad donde se celebrará el evento, el lugar de inscripción, las características de accesibilidad y medios de transporte para llegar, los tipos de público y de evento, y las fechas y horas de inicio y fin de las actividades. También se muestran las propiedades que se pueden utilizar de otros vocabularios importados, como la Event ontology y schema.org/Event

También se pueden utilizar, y se recomienda utilizar, propiedades generales de Dublin Core (por ejemplo, dct:title y dct:identifier) y otras propiedades generales como rdfs:label, rdfs:comment, owl:sameAs, etc.
